# 官方 3.5.6 复刻对比简报

更新时间：2026-05-29

对比来源：`E:\test\tlcq\_____3.5.6` 官方反编译结果。

## 结论

当前项目已经把“本地 BLE/QGJ 控车 + 官方账号登录 + 官方车辆列表 + 基础云端控车”做成了可用子集。

但官方 3.5.6 是完整平台型 App，包含绑定、地图轨迹、BMS、NFC、消息、家庭共享、充电站、支付、售后、多车系硬件、OTA 等大量生态功能。当前项目还不能算完整复刻。

大致复刻度：

- 按完整官方 App 算：约 **25%-35%**
- 按控车工具算：约 **65%-75%**

## 优先补什么

优先做低风险、只读、能提升真实体验的功能：

1. 官方 BMS/电池详情只读。
2. 官方消息列表和消息详情只读。
3. 官方轨迹、围栏数据只读。
4. 官方 NFC 钥匙列表只读。
5. 官方车库/绑定信息只读。

## 暂时不要做

这些先保持禁用，等反编译证据 + 真机验证 + 失败回滚方案都有了再说：

- NFC 真加钥匙、删除钥匙。
- 真正 OTA。
- 密码解锁读取/写入。
- 自动锁车、电子龙头锁、边撑、坐垫、侧翻检测等高级 ECU 写入。
- 支付、续费、充电桩交易、保险、商城、金融。

## 判断

当前方向没问题：先把控车和官方云只读数据做稳，再逐步补生态。下一步不要继续堆页面壳，应该优先接官方 BMS、消息、轨迹、围栏、NFC 列表这些真实只读数据。
