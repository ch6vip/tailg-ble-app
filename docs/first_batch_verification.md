# 第一批功能真机验证清单

目标：验证本地车辆绑定/多车管理、真实位置页、QGJ 高级设置、电池/BMS 详情、OTA 前置检测在实际车辆上的表现。

## 环境准备

- Android 设备已安装最新 debug APK
- 手机蓝牙、定位服务已开启
- App 已授予蓝牙扫描、蓝牙连接、定位权限
- 车辆处于可被蓝牙扫描和连接的状态

## 本地车辆绑定/多车管理

- 在“扫描”页可以发现车辆蓝牙设备
- 点击设备后可以连接成功
- 连接成功后设备自动加入“设置 > 我的车库”
- 车库中显示车辆名称、蓝牙 ID、协议类型、最后连接时间
- 可以编辑车辆名称
- 可以为 QGJ 车辆编辑 ECU 登录密码和用户 UID，清空后回到默认 0/0
- 多辆车存在时可以切换默认车辆
- 删除默认车辆后，车库会自动选择下一辆车或清空默认车辆
- 重新打开 App 后，车库数据仍然存在
- 自动连接开启时，App 优先扫描并连接默认车辆

## 真实位置页

- 进入“车辆位置”页后，点击刷新会触发系统定位权限请求
- 授权后可以显示坐标、定位精度和记录时间
- 点击复制按钮后，剪贴板内容为当前坐标
- 点击打开地图后，可以跳转到外部地图/浏览器查看坐标
- 已授权定位后，连接成功会静默记录默认车辆最后位置
- 已授权定位后，解锁/设防/寻车/开座桶/通电/断电成功会更新最后位置
- 拒绝定位权限时，页面给出可理解的错误提示

## QGJ 高级设置

- 车辆连接并进入 ready 状态后，“车辆设置”页可操作
- 点击右上角刷新按钮后，应通过官方 QGJ 命令读取光感、声音、震动灵敏度
- 点击右上角刷新按钮后，不应把 fcc1 的 ECU 功能位误显示为灯光状态
- 光感开关可以读写，命令为 `0x2411/0x2410`
- 启动、熄火、上锁、解锁、速度提示音可以读写，命令为 `0x2420/0x2421`
- 防盗灵敏度可以在“关闭 / 低 / 中 / 高”四档切换，对应写入 `0/15/50/85`
- 骑行模式超能跑/全速跑/超速跑切换后，设备返回确认且 UI 显示正确
- 切换骑行模式后再次读回 fcc1，不应清除其他 ECU 功能位
- 未确认的前灯、转向灯、蜂鸣器独立音量不应向设备写入
- 不支持的车辆或缺少 fcc1 特征时，页面能提示失败原因
- 进入“车辆设置 > 高级设置只读”后，只能看到刷新和复制结果，不应出现高级写入开关
- 高级只读刷新后，诊断日志应包含每条 GET 的命令名、cmdId、请求 payload、响应 payload 和解析值
- 高级只读复制结果应包含车辆连接状态、设备名、Remote ID 和当前解析结果
- 密码解锁 GET 不应自动触发，OTA 模式入口不应触发

## 电池/BMS 详情

- 车辆 ready 后，心跳数据能刷新电池详情页
- 电量百分比、电压、温度、信号值与控制页一致
- 低电量或故障位出现时，状态和故障列表显示正确
- 无心跳数据时，页面显示占位值而不是崩溃
- BMS 详情区应展示官方字段结构：SOC、SOH、当前电压、充电状态、容量、电流、环境温度、循环次数、电池温度、电池类型、硬件版本、软件版本
- 仅 feb3 已确认来源的 SOC、电压、温度显示实际值，其余字段显示“待读取”
- BMS 详情区不应出现校准电池、换电池、电池升级等会触发服务端或 OTA 的操作
- 页面说明应明确字段结构已预留，未确认读取来源的字段不伪造数据

## OTA 前置检测

- 未连接车辆时，页面提示等待可检测设备
- 连接 QGJ 车辆时，可以识别 feb0 服务
- 连接标准车辆时，可以识别 fee5 服务
- 存在 fcc0 时，页面显示扩展服务存在
- 如果设备提供 180A 服务，可以读取型号、固件版本 2A26、制造商 2A29
- 如果设备不提供 180A 服务，页面显示未读取到而不是报错
- 页面全程不执行任何 OTA 写入、擦写或升级动作

## 官方账号和云端控车

- 进入“设置 > 官方账号”后，未登录状态显示手机号、短信验证码和登录按钮
- 输入用户自己的官方账号手机号后，可以请求短信验证码
- 验证码请求失败时，页面显示明确错误，不记录手机号明文到普通日志
- 输入正确验证码后，可以登录官方账号
- 登录成功后，页面显示脱敏手机号和车辆列表
- token 失效或官方接口返回认证失败时，车辆列表刷新和云端控车都会清理官方登录态并提示重新登录
- 升级旧版本后，已保存的官方 token/手机号会迁移到安全存储，普通偏好里不再保留 `official_cloud_token` / `official_cloud_phone`
- 官方车辆列表展示车辆名、在线状态、电量、电压、设防、ACC、蓝牙名和蓝牙 MAC
- 官方车辆详情展示车辆照片、昵称、车架号、官方 IMEI/GPS IMEI、命令 IMEI、在线、电量、电压、里程和经纬度
- IMEI、GPS IMEI、命令 IMEI、蓝牙 MAC 在详情页应脱敏展示
- 官方车辆详情可以执行“云端自检”，应记录 `app/device/cmd/status` 的成功/失败和返回字段摘要
- 官方车辆可以手动关联本地 BLE 车辆，关联后本地默认车辆同步切换到该 BLE 车辆
- 控制页可以进入“控车通道”，并在 `自动 / BLE / 官方云端` 三种通道之间切换
- `BLE` 通道只在本地蓝牙 ready 时允许控车
- `官方云端` 通道只在官方账号已登录且已选择车辆时允许控车
- `自动` 通道在官方车辆已关联本地 BLE 车辆时，只允许关联的本地默认车辆 ready 后优先走 BLE；未关联时保持 BLE ready 优先；BLE 不可用且官方车辆可用时走官方云端
- 官方云端控车支持设防、解防、上电、断电、寻车、开坐垫
- 每条官方云端命令应显示成功或失败反馈，失败后按钮恢复
- 官方云端控车失败不应影响 BLE 本地控车
- 普通日志不应输出官方 token、手机号、IMEI 明文
- 测试报告需记录：官方账号是否登录成功、车辆数量、车辆 modelType、命令使用的是 IMEI 还是 GPS IMEI、云端自检返回、每条命令返回结果、控车前后车辆实际状态
- 诊断报告应包含 `Official Cloud` 段，展示脱敏手机号、token 是否存在、车辆数量、当前通道、选中车辆、modelType、命令 IMEI、设防/ACC/电量/电压和自检/控车日志
