## 三、Sprint 1 — P0 热修（1-2 周，8 项）

### P0-1: `_disconnectHandled` 重连后永不复位（已完成）
- **原始定位**：`lib/ble/connection_manager.dart:69`（声明）、`:740-741`（守卫）、`:806-808`（重连成功路径缺复位）
- **原始根因**：`connect()` 在 line 187 复位该标志，但 `_attemptReconnect()` 成功 return 前（line 806-808）未复位 `_disconnectHandled`，导致重连成功后再次断连时 `_onDisconnected()` 直接 return。
- **现状**：
  1. `_attemptReconnect()` 成功路径在记录 `重连成功` 前执行 `_disconnectHandled = false;`。
  2. 守卫逻辑已收敛到 `_markDisconnectHandled()`，`_onDisconnected()` 通过 `if (!_markDisconnectHandled()) return;` 防止重入。
- **验收证据**：`test/connection_manager_reconnect_test.dart` 覆盖首次断连标记、重入拦截、重连成功复位以及多轮断连/重连循环。
- **风险/回滚**：低风险；回滚即删除一行赋值。

## 四、Sprint 2 — P1 系统性修复（2-3 周，20 项，按系统性问题归类）

### S1: 单例生命周期混乱（P1-3）
**统一策略**：定义两个明确语义接口：`dispose()` = 永久销毁（仅 App 退出调用）；`resetForTest()` = 测试间复位状态但保留 StreamController。所有单例服务实现 `LifecycleService` 抽象接口。
- `service_locator.dart:112-134` 的 `dispose()` 内对单例服务的 `dispose()` 调用全部移除，改为仅在 `AppServices.reset()`（测试用）调 `resetForTest()`
- `connection_manager.dart:965 dispose()` 仅关闭 controller，不复位单例标志——新增 `resetForTest()` 复位 `_state/_disconnectHandled/_reconnecting` 但不 close controller
- `AppServices.dispose()` 仅在 `AppLifecycleState.detached` 调用

### S2: 超时分层不一致（P1-1, P1-2）
> 2026-07-03 更新：P1-2 connected→ready watchdog 已由 `_readyWatchdog` 覆盖，`connection_manager_state_test.dart` 验证超时断连与 ready 后解除超时。本节剩余风险聚焦连接总超时、GATT/Completer 全局超时分层和真实设备基线。

**统一策略**：引入 `lib/ble/ble_timeouts.dart` 集中定义分层超时常量，保证**外层超时 > 内层重试总耗时**。公式：`outer >= innerAttempts * (innerTimeout + retryDelay)`。
- `BleTimings.gattOperationTimeout` 当前定义 GATT 单操作 30s；`_connectDeviceWithRetry` 3 次 × connectTimeout → 外层 `connect()` 调用方超时需 ≥ 3×(connectTimeout+retryDelay)。在 `connect()` 顶部包 `Future.timeout(BleTimings.connectTotalTimeout)`
- 全局审计 `Completer`，确保每个都有 `.timeout()` 或在 `_completePendingOperations` 中被 complete。新增 `_disposeAllCompleters()` 在 dispose/reset 路径强制 complete

### S3: 空 catch 泛滥（P1-4，`catch (_)` 已清零）
**统一策略**：禁用 `catch (_) {}` 与 `catch (e) {}` 无处理模式。新增 lint `avoid_empty_catch`。所有 catch 必须至少 `_log.warning(...)` 或显式 `// ignore` 注释说明理由。
- **P1-4（已完成）**：`service_locator.dart` cleanup 已统一走 `_runCleanup()` 打印错误与 debug stack；`ConnectionManager._recoverFailedConnect()` 的 best-effort `disconnect()` 失败已改为 debug 级 BLE 日志；`DeviceInfoPage` 与 `OtaPrecheckPage` 的 GATT Device Information 字段读取失败已改为 debug 级操作日志；BLE parser、官云响应解码与诊断历史解析已移除 wildcard catch 变量，并由 `test_conventions_test.dart` 全局守卫 `lib/` 不再出现 `catch (_)`。

### S4: 状态机标志位复位不全（含 P0-1 衍生 + P2-6）
**统一策略**：将 `ConnectionManager` 形式化为状态机（见横向建议 7.4）。所有标志位（`_disconnectHandled/_reconnecting/_reconnectCancelled/_userDisconnected`）的复位收敛到 `_enterState(ConnectionState)` 单一入口，禁止散落赋值。
- `_disconnectHandled` 仅在进入 `connecting`/`reconnecting` 时复位
- `_reconnecting` 仅在进入 `reconnecting` 置 true、离开置 false
- `ProximityService` 竞态同步治理

### S5: 协议层无帧校验 + 近距解锁（P1-5, P1-6, P2-1）
- `lib/ble/qgj_protocol.dart` 增加 CRC8/CRC16 校验字段（向后兼容：旧帧无 CRC 时降级为长度校验 + warning log）

## 七、横向重构建议（跨 Sprint）

### 7.1 状态管理统一
**现状**：全项目仍以 setState/手写订阅为主；`_HomeBody` 的合并流已拆成局部 `ValueNotifier` + 独立 `StreamBuilder`，但更大范围的精准订阅仍未系统化。**建议**：引入 **Riverpod 2.x**（轻量、编译安全、可测试）。`ConnectionManager`/`VehicleStore`/`OfficialCloudService` 改为 `Notifier`/`AsyncNotifier`，`AppServices` 退化为 `ProviderContainer`。UI 用 `ref.watch` 精准订阅，消除整页重建。迁移路径：Sprint 2 先包装为 `Provider`，Sprint 3 逐步改 `Notifier`。

### 7.2 Token 体系重建
路径：`docs/design_system.md` + 当前 `lib/theme/` token 盘点 → `lib/theme/app_tokens.dart` 定义语义 token（`AppTokens.surface`/`onSurface`/`primary` 等）→ `AppColorsLight`/`AppColorsDark` 实现具体色值 → `ThemeExtension<AppTokens>` 注入 → `MaterialApp` 接 `darkTheme`。UI 全部读 `Theme.of(context).extension<AppTokens>()!`，禁止直引 `AppColors.xxx`。

### 7.3 单例生命周期统一
定义 `abstract class LifecycleService { void resetForTest(); Future<void> dispose(); }`。`resetForTest` 仅复位状态字段不关 StreamController；`dispose` 仅在 App 退出调。`AppServices.reset()` 调 `resetForTest`，`AppServices.dispose()` 在 `WidgetsBindingObserver.didChangeAppLifecycleState(detached)` 调。

### 7.4 BLE 状态机形式化
`ConnectionManager` 引入显式状态机：
```
状态: disconnected | connecting | connected | ready | reconnecting
事件: connectRequested | connectionEstablished | tokenReceived | disconnected | reconnectTimeout | userDisconnect
转移表: (disconnected,connectRequested)->connecting | (connecting,connectionEstablished)->connected | (connected,tokenReceived)->ready | (ready,disconnected)->reconnecting | (reconnecting,reconnectTimeout)->disconnected | (*,userDisconnect)->disconnected
```
所有 `_setState` 调用改为 `_transition(event)`，转移函数内统一复位标志位。

### 7.5 测试金字塔补全
- **单元**：`ble/parser_test.dart`（边界已补，CRC 待 S5）、`qgj_protocol_test.dart`、`vehicle_store_test.dart`（已存在，加深）
- **集成**：`integration_test/connection_flow_test.dart`（mock `flutter_blue_plus` 设备，验证 connect→ready→disconnect→reconnect 全链路）
- **E2E**：`integration_test/app_test.dart` 用 golden test 验证关键页面渲染
- 工具：引入 `mockito` + `build_runner`，`flutter_blue_plus` 用 `FakeBluetoothDevice` mock
