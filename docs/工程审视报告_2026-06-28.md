# Tailg BLE App 工程审视与分阶段重构方案

> 审视范围：全项目（lib/ble、lib/services、lib/models、lib/pages、lib/widgets、lib/theme、test/、.github/workflows、平台配置）
> 审视维度：长期维护性 / 扩展性 / 稳定性 / UX / 生产环境适配
> 交付：P0~P2 完整方案（约 40 项），五维度均衡，分阶段路线图
> 审视日期：2026-06-28
> 现状更新：2026-07-03 复核 `.github/workflows/`，P0-7、P0-8、P1-16 已完成；`build.yml` 不再监听 `v*` tag，`release.yml` 是唯一 Release 发布入口且包含 format/analyze/test 门禁。

## 一、执行摘要

项目整体健康度处于**"可运行但工程债务高"**状态：核心 BLE 连接链路可用，但存在多处会在生产环境显形的状态机复位缺陷、文档与 CI 配置背离、UI 层重建范围失控。

**最紧迫的三个风险：**
1. **二次断连无响应**——`_disconnectHandled` 重连成功后不复位，用户拔蓝牙后 App 假死（P0-1）
2. **暗色主题完全不可用**——`AppColorsDark` 已定义却被 `main.dart:228` 硬编码 `ThemeMode.light` 旁路（P0-2）
3. **CI 双 Release 并发 + 文档虚构（已完成）**——原风险为 `v*` tag 同时触发 build.yml 与 release.yml；当前 `build.yml` 已移除 tag 触发，AGENTS 与 CI 指南已对齐真实工作流（P0-7/8）

**重构总策略：先止血、再治本、后保洁。**

## 三、Sprint 1 — P0 热修（1-2 周，8 项）

### P0-1: `_disconnectHandled` 重连后永不复位
- **定位**：`lib/ble/connection_manager.dart:69`（声明）、`:740-741`（守卫）、`:806-808`（重连成功路径缺复位）
- **根因**：`connect()` 在 line 187 复位该标志，但 `_attemptReconnect()` 成功 return 前（line 806-808）未复位 `_disconnectHandled`，导致重连成功后再次断连时 `_onDisconnected()` 直接 return。
- **修复方案**：
  1. 在 `_attemptReconnect()` 成功路径（line 808 `_log.ble('重连成功')` 之前）插入 `_disconnectHandled = false;`
  2. 将守卫逻辑收敛到私有方法 `bool _markDisconnectHandled() { if (_disconnectHandled) return false; _disconnectHandled = true; return true; }`，`_onDisconnected` 改为 `if (!_markDisconnectHandled()) return;`
- **验收标准**：新增测试 `test/connection_manager_reconnect_test.dart`：模拟 ready → 断连 → 重连成功 → 再次断连，断言第二次 `_onDisconnected` 执行且 stateStream 发出 `reconnecting`。
- **风险/回滚**：低风险；回滚即删除一行赋值。

### P0-2: 暗色主题被硬编码旁路
- **定位**：`lib/main.dart:228`（`themeMode: ThemeMode.light`）、`lib/theme/app_colors.dart`（`AppColorsDark` 定义但未接线）
- **修复方案**：
  1. `MaterialApp` 增加 `darkTheme: ThemeData(brightness: Brightness.dark, colorScheme: ColorScheme.fromSeed(seedColor: AppColors.primary, brightness: Brightness.dark))`
  2. `themeMode` 改为 `ThemeMode.system`（跟随系统）
  3. `AppColors` 改为依据 `Theme.of(context).brightness` 的适配器，而非静态 const
- **验收标准**：系统切暗色后页面背景、卡片、文字均切换；新增 `test/theme_test.dart` 验证两模式正确。
- **风险/回滚**：中风险（硬编码色散落，暗色下对比度可能不足，Sprint 3 统一 Token 后做对比度审计）；回滚为 `ThemeMode.light`。

### P0-3: `_HomeBody` dispose 竞态
- **定位**：`lib/pages/control_page.dart:162-176`
- **根因**：`dispose()` 调用 `_cancelSubscriptions()`（async Future）但不 await，随后 `_controller?.close()` 可能先执行，触发 `onCancel` 再次进入竞态。
- **修复方案**：改为同步取消 + 标志位保护：
  ```dart
  bool _disposed = false;
  @override
  void dispose() {
    _disposed = true;
    _subConn?.cancel();
    _subVehicles?.cancel();
    _subCloud?.cancel();
    _controller?.close();
    super.dispose();
  }
  ```
  `_createCombinedStream` 内 `emit()` 与监听回调首行加 `if (_disposed) return;`。
- **验收标准**：快速切 tab 不抛 `StateError`、不泄漏订阅；`pumpWidget`+`pump` 循环 20 次无异常。
- **风险/回滚**：低。

### P0-4: 单 StreamBuilder 触发整页重建
- **定位**：`lib/pages/control_page.dart:180-241`
- **根因**：三流（connectionManager.state / vehicleStore.vehicles / officialCloudService.state）合并后单 builder 重建全部子树。
- **修复方案**：拆分为三个独立 `StreamBuilder`，每个只订阅各自流。`_HomeBody.build` 返回 `Column` 含三个独立 builder。外层 `AnimatedSwitcher` 仅依赖 `showUnboundHome` 派生流。
- **验收标准**：仅连接态变化时 `_RidingModeSelector` 不重建（用 `debugRepaintRainbowEnabled` 验证）；滚动时 <16ms。
- **风险/回滚**：中（需保证三流初值同步）；回滚为单 builder。

### P0-5: `location_page` 空 setState 重建含 flutter_map 整页
- **定位**：`lib/pages/location_page.dart`
- **修复方案**：map widget 用 `const` + `RepaintBoundary` 隔离；移除空 `setState`，改为局部 `ValueNotifier` 驱动仅需要刷新的子树。
- **验收标准**：`flutter run --profile` + Performance overlay 显示地图拖动时无额外 rebuild。
- **风险/回滚**：低。

### P0-6: 绕过 service_locator 单例
- **定位**：多处直接 `XxxService()` 而非经 `AppServices.instance`
- **修复方案**：全局搜索 `Service()` 直接构造，全部替换为 `main.dart` 顶层 getter（已委托 `AppServices.instance`）。新增 lint + code review 守卫。
- **验收标准**：`Grep` 搜索 `lib/pages`、`lib/widgets` 内 `Service()` 直接构造为 0；`AppServices.override` 在测试中能完全替换依赖。
- **风险/回滚**：低。

### P0-7: AGENTS.md 虚构 ci.yml（已完成）
- **原始定位**：`AGENTS.md:34-46`（已验证 `.github/workflows/` 实际只有 `build.yml` 和 `release.yml`）
- **现状**：`AGENTS.md` 与 `docs/github_actions_guide.md` 已改为真实两工作流；`build.yml` 描述为 CI + APK artifact，`release.yml` 描述为唯一 Release 发布入口。
- **验收证据**：`rg -n "ci\\.yml" AGENTS.md` 无结果；`Get-ChildItem -Name .github\workflows` 仅返回 `build.yml`、`release.yml`。
- **风险/回滚**：无（纯文档）。

### P0-8: build.yml 与 release.yml 双 Release 并发（已完成）
- **原始定位**：`build.yml` 与 `release.yml` 都监听 `v*` tag，且都可能创建 GitHub Release。
- **现状**：`build.yml` 只监听 `master`/`develop` push、PR 和手动触发；`release.yml` 监听 `v*` tag 和手动触发，并且是唯一调用 `softprops/action-gh-release@v2` 的工作流。
- **验收证据**：`rg -n "tags|action-gh-release" .github/workflows/build.yml .github/workflows/release.yml` 只在 `release.yml` 命中 tag 与 Release action。
- **剩余验证**：真实推送 `v*` tag 后确认只生成 1 个 GitHub Release。
- **风险/回滚**：中（需测试 tag 推送）；回滚恢复双触发。

## 四、Sprint 2 — P1 系统性修复（2-3 周，20 项，按系统性问题归类）

### S1: 单例生命周期混乱（P1-3）
**统一策略**：定义两个明确语义接口：`dispose()` = 永久销毁（仅 App 退出调用）；`resetForTest()` = 测试间复位状态但保留 StreamController。所有单例服务实现 `LifecycleService` 抽象接口。
- `service_locator.dart:112-134` 的 `dispose()` 内对单例服务的 `dispose()` 调用全部移除，改为仅在 `AppServices.reset()`（测试用）调 `resetForTest()`
- `connection_manager.dart:965 dispose()` 仅关闭 controller，不复位单例标志——新增 `resetForTest()` 复位 `_state/_disconnectHandled/_reconnecting` 但不 close controller
- `AppServices.dispose()` 仅在 `AppLifecycleState.detached` 调用

### S2: 超时分层不一致（P1-1, P1-2）
**统一策略**：引入 `lib/ble/ble_timeouts.dart` 集中定义分层超时常量，保证**外层超时 > 内层重试总耗时**。公式：`outer >= innerAttempts * (innerTimeout + retryDelay)`。
- `connection_manager.dart:127` GATT 单操作 30s；`_connectDeviceWithRetry` 3 次 × connectTimeout → 外层 `connect()` 调用方超时需 ≥ 3×(connectTimeout+retryDelay)。在 `connect()` 顶部包 `Future.timeout(BleTimings.connectTotalTimeout)`
- 全局审计 `Completer`，确保每个都有 `.timeout()` 或在 `_completePendingOperations` 中被 complete。新增 `_disposeAllCompleters()` 在 dispose/reset 路径强制 complete

### S3: 空 catch 泛滥（P1-4，14+ 处）
**统一策略**：禁用 `catch (_) {}` 与 `catch (e) {}` 无处理模式。新增 lint `avoid_empty_catch`。所有 catch 必须至少 `_log.warning(...)` 或显式 `// ignore` 注释说明理由。
- `connection_manager.dart:872`（`_recoverFailedConnect`）、`:987`（dispose disconnect）、`service_locator.dart:90-108`（6 处 reset）全部改为 `catch (e, st) { _log.warning('...', e, st); }`
- `service_locator.dart` 的 reset 改为收集错误并一次性 rethrow 或 log

### S4: 状态机标志位复位不全（含 P0-1 衍生 + P2-6）
**统一策略**：将 `ConnectionManager` 形式化为状态机（见横向建议 7.4）。所有标志位（`_disconnectHandled/_reconnecting/_reconnectCancelled/_userDisconnected`）的复位收敛到 `_enterState(ConnectionState)` 单一入口，禁止散落赋值。
- `_disconnectHandled` 仅在进入 `connecting`/`reconnecting` 时复位
- `_reconnecting` 仅在进入 `reconnecting` 置 true、离开置 false
- `ProximityService` 竞态同步治理

### S5: 协议层无帧校验 + 近距解锁（P1-5, P1-6, P2-1）
- `lib/ble/qgj_protocol.dart` 增加 CRC8/CRC16 校验字段（向后兼容：旧帧无 CRC 时降级为长度校验 + warning log）
- **P1-5**：`proximity_service.dart` 解锁前断言 `location.accuracy <= 30m`，否则降级为"仅提示不解锁"并 log
- **P1-6**：`android/app/src/main/AndroidManifest.xml` 移除过度严格的 `BLUETOOTH_CONNECT` 运行时强制，按 `flutter_blue_plus` 文档最小集声明

### UI 系统性问题（P1-8 ~ P1-15）

## 七、横向重构建议（跨 Sprint）

### 7.1 状态管理统一
**现状**：全项目 setState，`_HomeBody` 手写流合并。**建议**：引入 **Riverpod 2.x**（轻量、编译安全、可测试）。`ConnectionManager`/`VehicleStore`/`OfficialCloudService` 改为 `Notifier`/`AsyncNotifier`，`AppServices` 退化为 `ProviderContainer`。UI 用 `ref.watch` 精准订阅，消除整页重建。迁移路径：Sprint 2 先包装为 `Provider`，Sprint 3 逐步改 `Notifier`。

### 7.2 Token 体系重建
路径：`design_v2/` 设计 token → `lib/theme/app_tokens.dart` 定义语义 token（`AppTokens.surface`/`onSurface`/`primary` 等）→ `AppColorsLight`/`AppColorsDark` 实现具体色值 → `ThemeExtension<AppTokens>` 注入 → `MaterialApp` 接 `darkTheme`。UI 全部读 `Theme.of(context).extension<AppTokens>()!`，禁止直引 `AppColors.xxx`。

### 7.3 单例生命周期统一
定义 `abstract class LifecycleService { void resetForTest(); Future<void> dispose(); }`。`resetForTest` 仅复位状态字段不关 StreamController；`dispose` 仅在 App 退出调。`AppServices.reset()` 调 `resetForTest`，`AppServices.dispose()` 在 `WidgetsBindingObserver.didChangeAppLifecycleState(detached)` 调。

### 7.4 BLE 状态机形式化
`ConnectionManager` 引入显式状态机：
```
状态: disconnected | connecting | connected | ready | reconnecting
事件: connectRequested | connectionEstablished | tokenReceived | disconnected | reconnectTimeout | userDisconnect
转移表: (disconnected,connectRequested)->connecting | (connecting,connectionEstablished)->connected | (connected,tokenReceived)->ready | (ready,disconnected)->reconnecting | (reconnecting,reconnectTimeout)->disconnected | (*,userDisconnect)->disconnected
```
所有 `_setState` 调用改为 `_transition(event)`，转移函数内统一复位标志位。

### 7.5 测试金字塔补全
- **单元**：`ble/parser_test.dart`（边界/CRC）、`qgj_protocol_test.dart`、`vehicle_store_test.dart`（已存在，加深）
- **集成**：`integration_test/connection_flow_test.dart`（mock `flutter_blue_plus` 设备，验证 connect→ready→disconnect→reconnect 全链路）
- **E2E**：`integration_test/app_test.dart` 用 golden test 验证关键页面渲染
- 工具：引入 `mockito` + `build_runner`，`flutter_blue_plus` 用 `FakeBluetoothDevice` mock
